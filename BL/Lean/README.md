# Lean verification workspace

This folder is the staged Lean workspace for the formalization of
`../Arxiv_Split/main.tex`.

The intended order is:

1. `L2Counterexample/Basic.lean`
2. `L2Counterexample/Bump.lean`
3. `L2Counterexample/Potential.lean`
4. `L2Counterexample/Normalization.lean`
5. `L2Counterexample/TestFunction.lean`
6. `L2Counterexample/OneDimensional.lean`
7. `L2Counterexample/HigherDimensional.lean`

The dependency audit in `../Arxiv_Deps/big_tasks.md` is the checklist for the
large formalization tasks. The files here are deliberately skeletal: they set up
the module structure without adding axioms or `sorry`-based theorem placeholders.

When Lean and Lake are installed, the usual workflow will be:

```bash
cd BL/Lean
lake update
lake exe cache get
lake build
```

The toolchain is pinned to `leanprover/lean4:v4.30.0-rc2`, matching the
mathlib manifest generated by `lake update`.
